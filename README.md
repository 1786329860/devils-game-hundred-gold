# 《恶魔游戏：百金争夺》

这是一个使用 **Godot 4.6 标准版（GDScript）**制作的完整 Windows 心理博弈游戏。项目包含单机效用 AI、每日挑战、契约变体、长期成长，以及服务器权威的 1v1 联机对战。项目附带可直接运行的便携版、完整源工程、云端服务、教程、无障碍设置、中文台词、程序化音效和自动化验收工具。

当前修复版：`1.1.1`。该版本修复联机胜负回合服务端异常断线，并加入客户端 WebSocket 心跳。

## 直接运行

进入 `build` 文件夹，双击 `DevilsGame.exe`。`DevilsGame.pck` 必须和它放在同一目录。

最低建议环境：Windows 10/11、支持 OpenGL 3.3 的显卡、1280×720 或更高分辨率。

## 游戏内容

- 1v1、100 金币池、五枚一次性棋子。
- 国王 > 将军 > 骑士 > 士兵 > 平民；平民反杀国王。
- 一枚钻石，支持明牌升级和暗牌升级。
- 任意展示名称伪装。
- 双方共享两次、70% 准确率的模糊读牌。
- 大额平局惩罚、失忆 Debuff、二级平民绝杀。
- 保守、激进、欺诈、套路四种人格，入门、进阶、地狱三档难度。
- 平民钓鱼、欺诈耗牌、钻石骗招、将军突击、残局收割五套多轮剧本。
- 156 条配置化中文人格台词。
- 六种恶魔契约变体、每日固定种子挑战、等级、契印、连胜和八项成就。
- 公网匹配、六位私人房间码、双阶段密封意图、共享读牌抢用与改招。
- 90 秒断线重连、主动认输、云端 SQLite 战绩、Elo 评级和赛季排行榜。

## 联机服务

- 游戏地址：`wss://tucao.aixiaolv.icu/ws`
- 健康状态：`https://tucao.aixiaolv.icu/health`
- 排行榜：`https://tucao.aixiaolv.icu/leaderboard`

联机模式采用两人对战，不开发三人以上同桌。百金池、共享读牌、一次性棋子和“读当前暗牌”都以一对一零和推理为核心；多人同桌会引入围攻、串通和过长等待，削弱游戏特色。服务端协议保留观战和赛事扩展空间。

## AI 架构

核心位于 `scripts/core/utility_ai.gd`，决策顺序为：

1. 从公开历史建立对手模型和置信度。
2. 枚举合法棋子、升级和展示名，最多 75 项。
3. 计算 F1-F10、人格修正、特殊策略、剧本修正和难度噪声。
4. 选择最高效用行动并生成逐候选日志。

玩家手牌和当前选择不会出现在 AI 公开状态中。AI 只能使用已揭示记录、金币、共享资源和合法模糊读牌。

## 打开源工程

1. 安装 Godot 4.6.3 标准版或兼容的 4.6.x 标准版。
2. 在项目管理器中导入本目录的 `project.godot`。
3. 按 `F6` 或右上角运行按钮启动主场景。

项目不依赖 .NET、C#、第三方插件或外部运行库。

## 策划调参

- `data/ai_config.json`：4 种人格、3 档难度、12 个组合、5 套剧本和全局阈值。
- `data/dialogue_zh_cn.json`：中文台词、事件、优先级、冷却和条件。
- `scripts/core/game_rules.gd`：棋子克制、升级、金币和合法性规则。

人格八项直接权重之和必须为 `1.0`。难度只改变噪声、历史深度、读牌阈值、伪装精度和连招概率，不授予隐藏信息。

## 设计补全

原文同时给出“五枚一次性棋子”和“预计 5-8 轮”。两者无法在无额外棋子的前提下同时成立，因此成品采用最多五个有效回合，并按双方剩余棋子动态生成下一轮金币。平局金币和无棋可出时的剩余金币计入“浪费金币”，所以 100 金币始终守恒为：玩家金币 + AI 金币 + 浪费金币。

```
│   ├── core/            # 核心逻辑（AI、规则、联机、存档）

├── server/              # Python WebSocket 联机服务器

│   ├── game_rules.py    # 服务端规则验证
│   ├── storage.py       # SQLite 存储层
│   ├── deploy/          # Nginx 部署配置
│   ├── Dockerfile       # Docker 构建
│   └── docker-compose.yml
├── data/                # 游戏数据（AI 配置、台词 JSON）
├── assets/              # 美术和音频资源
├── build/               # 构建说明和启动脚本
├── docs/                # 测试报告、架构文档、新手指南

```
