# AI 效用系统 — 完整设计文档

> 本文档定义《恶魔游戏：百金争夺》中 AI 对手的完整决策架构。
> 目标：让 AI 在不同人格和难度下表现出"像真人一样"的心理博弈行为，而非机械式最优解。

---

## 一、系统架构总览

### 1.1 核心思路

AI 的每一次决策都遵循同一套流程：

1. **收集当前局势信息**（GameState）
2. **枚举所有合法行动**（Action Candidates）
3. **为每个行动计算综合效用分数**（Utility Score）
4. **加入人格偏差和难度噪声**
5. **选择最终行动**

这不是一个有限状态机（FSM），而是一个效用系统（Utility System）。状态机的问题是行为可预测——玩家一旦摸清状态转换规则就能稳赢。效用系统通过多维度加权评分产生决策，同一个局势在不同人格、不同轮次、不同历史记录下会产出不同选择，让 AI 行为更难被模式化。

### 1.2 决策时机

AI 每轮需要做出以下决策（按顺序）：

| 决策编号 | 决策内容 | 可选范围 |
|---------|---------|---------|
| D1 | 是否使用读牌机会 | 使用 / 不使用 |
| D2 | 选择出战棋子 | 当前所有未失效的棋子 |
| D3 | 是否使用钻石升级 | 不使用 / 明牌升级 / 暗牌升级 |
| D4 | 如果升级，升级哪枚棋子 | 当前选择的出战棋子（只有一个） |
| D5 | 设置伪装名称 | 任意棋子名称（含自身真实名称=不伪装） |

注意：D1 先于 D2 决策，因为读牌获取的信息会影响后续选择。D3-D5 可以在同一个评估流程中联合决策。

### 1.3 信息边界（极其重要）

**AI 只能访问公开信息，绝不能窥探玩家手牌。** 这是公平性的底线。

**AI 可见的信息：**
- 自己的手牌（棋子列表、钻石状态）
- 双方当前金币数
- 双方已打出的棋子（及打出时的真实状态）
- 双方已使用的读牌次数
- 当前轮次的金币数量
- 当前轮次编号
- 全局总金币数（100）
- 历史出牌记录（受失忆 Debuff 影响可能被擦除）

**AI 不可见的信息：**
- 玩家当前手牌
- 玩家的伪装设置
- 玩家的钻石升级选择
- 玩家当前的决策过程

## 二、游戏状态数据结构

### 2.1 GameState（对局全局状态）

```
GameState {
    currentRound: int           // 当前轮次（从1开始）
    totalRounds: int            // 预估总轮次（5-8轮，由金币分布决定）
    totalGoldPool: int          // 全局总金币（固定100）
    remainingGold: int          // 尚未分配的金币数
    currentRoundGold: int       // 本轮金币数
    isLargeRound: bool          // 本轮是否大额轮（≥30枚）

    ai: PlayerState             // AI 自身状态
    opponent: PlayerState       // 玩家状态（仅公开信息）

    history: RoundRecord[]      // 历史出牌记录（可能被失忆 Debuff 擦除部分）
    aiReadingsLeft: int         // AI 剩余读牌次数（全局共享2次）
    opponentReadingsLeft: int   // 玩家剩余读牌次数
    amnesiaActive: bool         // 失忆 Debuff 是否已触发

    lastReadingResult: ReadingResult?  // 最近一次读牌返回的信息（如有）
}
```

### 2.2 PlayerState（单方状态）

```
PlayerState {
    gold: int                   // 当前累计金币
    availablePieces: Piece[]    // 当前可用棋子列表
    diamondUsed: bool           // 钻石是否已使用
    // --- 以下字段 AI 对自身可见，对玩家仅可见公开部分 ---
    totalPiecesUsed: int        // 已使用的棋子数量
    highPiecesRemaining: int    // 剩余高阶棋子数（国王/将军/骑士）
}
```

### 2.3 Piece（棋子）

```
Piece {
    id: string                  // 唯一标识
    type: PieceType             // King / General / Knight / Soldier / Commoner
    rank: int                   // 战力等级（Commoner=1, Soldier=2, Knight=3, General=4, King=5）
    isAvailable: bool           // 是否可用
    // --- 以下仅在出牌时确定 ---
    diamondUpgrade: UpgradeType // None / OpenUpgrade / SecretUpgrade
    disguiseAs: PieceType?      // 伪装成什么（null=不伪装）
}
```

### 2.4 RoundRecord（单轮记录）

```
RoundRecord {
    roundNumber: int
    goldAmount: int
    aiPiece: PieceType          // AI 真实出牌
    aiUpgrade: UpgradeType
    aiDisguise: PieceType?
    opponentPiece: PieceType    // 玩家真实出牌
    opponentUpgrade: UpgradeType
    opponentDisguise: PieceType?
    winner: Side                // AI / Opponent / Draw
    goldWon: int                // 该轮赢取的金币（0=平局或对方赢）
}
```

## 三、效用评估核心

### 3.1 总效用公式

对于每个候选行动 `a`（即"出某棋子 + 某种升级方式 + 某种伪装"的组合），其总效用为：

```
U(a) = Σ [ w_i × F_i(a) ] + Noise(difficulty)
```

其中：
- `w_i` 是第 i 个评估因子的权重（由 AI 人格决定）
- `F_i(a)` 是第 i 个评估因子对行动 a 的评分（归一化到 0~1）
- `Noise(difficulty)` 是难度相关的随机扰动（入门难度噪声大，地狱难度噪声接近 0）

### 3.2 评估因子清单

共 10 个核心评估因子，每个因子独立评分后加权求和：

#### F1: 期望胜率（ExpectedWinRate）

**含义**：AI 评估自己出这枚棋子，赢的概率有多大。

**计算逻辑**：

```
F1(piece) = Σ [ P(opponent_plays_X) × Outcome(piece, X) ]
```

其中：
- `P(opponent_plays_X)` = AI 估算对手出棋子 X 的概率
- `Outcome(piece, X)` = 结算结果：赢=1，平=0.3，输=0

**对手出牌概率估算**（核心难点）：

AI 基于以下信息估算对手的出牌概率分布：

1. **剩余可用棋子**：排除已打出的棋子
2. **历史出牌模式**：对手在大额轮/小额轮偏好出什么棋子
3. **对手当前金币状态**：领先时可能保守，落后时可能激进
4. **当前轮金币数**：大额轮对手更可能出高阶棋子

具体算法：为每枚对手剩余棋子分配基础概率，再根据上述因素调整权重。基础概率默认均匀分布（每枚棋子等概率），然后根据历史数据和局势修正。

**考虑升级后的胜率**：如果 AI 选择升级棋子，需要用升级后的战力重新计算 Outcome。

#### F2: 金币期望值（GoldExpectedValue）

**含义**：这个行动预期能赢取多少金币。

```
F2(a) = F1(a) × (currentRoundGold / maxPossibleGold)
```

其中 `maxPossibleGold` 是单轮理论最大金币数（45）。归一化到 0~1。

#### F3: 棋子保留价值（PiecePreservationValue）

**含义**：这枚棋子留着以后用是不是更有价值。如果打出它会损失多少未来选项。

**计算逻辑**：

```
F3(piece) = 1 - (pieceIntrinsicValue × futureNeedFactor)
```

注意这里是反向的——保留价值越高，打出该棋子的效用越低。

`pieceIntrinsicValue` 棋子固有价值的参考值：

| 棋子 | 固有价值 | 说明 |
|------|---------|------|
| 国王 | 0.85 | 最高战力，但被平民克制 |
| 将军 | 0.70 | 次高战力，有平局封禁风险 |
| 骑士 | 0.55 | 无负面效果，万金油 |
| 士兵 | 0.25 | 低战力，前期消耗品 |
| 平民 | 0.50 | 常规最弱但弑王价值无限 |

`futureNeedFactor` 根据局势动态调整：

- AI 金币领先 → 保守策略 → 保留价值提升（factor 上升）
- AI 金币落后 → 激进策略 → 保留价值降低（factor 下降）
- 剩余轮次多 → 保留价值提升
- 剩余轮次少 → 保留价值降低（必须用掉了）

#### F4: 风险系数（RiskCoefficient）

**含义**：这个行动的风险有多大。高风险行动的效用应该被压低（除非 AI 人格偏好风险）。

```
F4(a) = 1 - riskLevel(a)
```

`riskLevel` 的构成：

- **被反杀概率**：对手克制 AI 所出棋子的概率
- **升级失败风险**：如果使用钻石升级，升级后仍可能输或平局（钻石浪费 + 金币收益为零）
- **将军封禁风险**：如果出将军且平局，将军永久作废
- **弑王风险**：如果出国王，被平民反杀的概率（对手有平民时此风险显著升高）

特殊规则：
- 国王的风险在对手仍有平民时大幅上升
- 将军的风险在大额轮上升（因为大额轮更容易平局，触发封禁）
- 二级国王的风险极高（只有二级平民能克制，但一旦被克制就是绝杀翻盘）

#### F5: 金币差压力（GoldDifferentialPressure）

**含义**：当前金币差距带来的紧迫感。落后越多，越需要冒险。

```
goldDiff = opponent.gold - ai.gold  // 正值=AI落后
pressure = goldDiff / totalGoldPool

F5 = clamp(0.5 + pressure, 0, 1)
```

这个因子会间接影响其他因子的权重解读：当 AI 落后很多时，即使保守型 AI 也会被迫考虑高风险选项。

#### F6: 轮次进度因子（RoundProgressionFactor）

**含义**：对局进行到不同阶段，策略重心不同。

```
progress = currentRound / estimatedTotalRounds

F6(a) 不是独立评分，而是作为其他因子的调节器：
- 前期（progress < 0.3）：试探为主，降低 F2 权重，提高 F3 权重
- 中期（0.3 ≤ progress < 0.7）：正常博弈
- 后期（progress ≥ 0.7）：决胜为主，提高 F2 权重，降低 F3 权重
```

#### F7: 伪装价值（DisguiseValue）

**含义**：对这个行动施加伪装的收益评估。

伪装有三种战术方向：

| 伪装类型 | 含义 | 价值来源 |
|---------|------|---------|
| 弱装强 | 低阶棋子伪装高阶 | 恐吓对手，逼对手保守 |
| 强装弱 | 高阶棋子伪装低阶 | 引诱对手出被克制的棋子 |
| 同阶互装 | 同阶棋子互相伪装 | 混淆对手判断 |

```
F7(disguise) = baseDisguiseValue × opponentCautionFactor × roundGoldFactor
```

- `baseDisguiseValue`：弱装强=0.6，强装弱=0.7（大额轮可达0.9），同阶互装=0.3
- `opponentCautionFactor`：对手越谨慎（保守型AI的对手更可能求平局），弱装强的效果越好
- `roundGoldFactor`：大额轮伪装价值更高，小额轮伪装意义不大

不伪装也有价值（避免被读牌发现伪装后反而暴露信息），所以"不伪装"也是一个合法选项，其效用 = 1 - F7。

#### F8: 钻石使用价值（DiamondUsageValue）

**含义**：是否在本轮使用钻石，以及明用还是暗用。

**不使用钻石**：效用 = 基础值（保留钻石的未来价值），通常为 0.5

**明牌升级**：
```
F8_open = (upgradeWinBoost × currentRoundGold / 45) - exposurePenalty
```
- `upgradeWinBoost`：升级后胜率提升量
- `exposurePenalty`：暴露升级信息的代价（对手知道了你的钻石已用）

**暗牌升级**：
```
F8_secret = (upgradeWinBoost × surpriseMultiplier × currentRoundGold / 45) - riskOfWaste
```
- `surpriseMultiplier`：突袭效果加成（通常 1.3-1.5）
- `riskOfWaste`：升级后仍输/平局的风险（钻石浪费 + 金币收益为零的惩罚）

暗牌升级的关键博弈：如果暗升平民为二级平民，伪装成普通平民引诱对手出国王——一旦成功就是"绝杀翻盘"。但如果对手没出国王，钻石就浪费了。

#### F9: 读牌价值评估（ReadingValue）

**含义**：是否在本轮消耗读牌机会。

```
F9_read = uncertaintyLevel × roundImportance × readingsRemainingFactor
```

- `uncertaintyLevel`：AI 对对手本轮出牌的不确定性（如果对手行为模式很清晰，此值低）
- `roundImportance`：本轮金币数越大，读牌价值越高
- `readingsRemainingFactor`：剩余读牌次数越少，使用越谨慎（最后一次读牌的价值阈值更高）

注意：读牌结果有 30% 误导概率。AI 需要在评估读牌结果时也考虑这个不确定性——对读牌信息的信任度应打折。

#### F10: 对手行为模式利用（OpponentPatternExploitation）

**含义**：根据对手的历史出牌模式，发现可利用的规律。

AI 维护一个简化的对手行为模型：

```
opponentTendencies = {
    largeRoundHighPieceRate: float    // 大额轮出高阶棋子的频率
    smallRoundPieceDistribution: {}   // 小额轮的出牌分布
    disguiseRate: float               // 使用伪装的频率
    diamondTiming: string             // 钻石使用倾向（前期/中期/后期/未使用）
    drawSeekingRate: float            // 主动求平局的频率
}
```

F10 的评分基于：当前行动是否针对对手的某个倾向做出了反制。

例如：如果对手在大额轮频繁出将军（largeRoundHighPieceRate 高），那么 AI 出平民（伪装成其他棋子）的价值会提升。

### 3.3 效用评估完整公式

```
U(action) =
    w1 × F1(piece)              // 期望胜率
  + w2 × F2(piece, gold)        // 金币期望值
  + w3 × F3(piece, state)       // 棋子保留价值（反向）
  + w4 × F4(action)             // 风险系数（反向）
  + w5 × F5(state)              // 金币差压力
  + w6 × F7(disguise)           // 伪装价值
  + w7 × F8(diamond)            // 钻石使用价值
  + w8 × F10(opponent, action)  // 对手模式利用
  + Noise(difficulty)
```

F6（轮次进度）和 F9（读牌价值）作为独立决策模块，不直接进入上述公式，而是在决策流程的更早阶段起作用（F9 决定是否读牌，F6 动态调整其他权重）。

## 四、AI 人格配置

### 4.1 权重配置表

四个人格使用同一套评估因子，但权重截然不同：

| 因子 | 保守型 | 激进型 | 欺诈型 | 套路型 |
|------|-------|-------|-------|-------|
| F1 期望胜率 | 0.20 | 0.15 | 0.12 | 0.18 |
| F2 金币期望值 | 0.08 | 0.25 | 0.12 | 0.15 |
| F3 棋子保留价值 | 0.20 | 0.05 | 0.08 | 0.12 |
| F4 风险系数 | 0.25 | 0.05 | 0.10 | 0.15 |
| F5 金币差压力 | 0.05 | 0.15 | 0.10 | 0.10 |
| F7 伪装价值 | 0.05 | 0.08 | 0.25 | 0.15 |
| F8 钻石使用价值 | 0.05 | 0.15 | 0.15 | 0.10 |
| F10 对手模式利用 | 0.12 | 0.12 | 0.08 | 0.05 |
| **合计** | 1.00 | 1.00 | 1.00 | 1.00 |

### 4.2 人格行为特征详解

#### 保守型 AI

**核心策略**：优先规避风险，宁可放弃金币也不愿损失高阶棋子。

**行为特征**：
- 大额轮倾向伪装棋子制造平局（而非争夺金币）
- 极少主动打出国王、将军等高风险棋子
- 钻石保留到非常后期才使用（甚至可能全局不使用）
- 前期频繁出士兵试探
- 对手出牌模式稳定后，会针对性规避

**额外行为修正**：
- 大额轮时 F4（风险系数）的权重临时上浮 30%
- 当 AI 金币领先时，F3（棋子保留价值）的权重临时上浮 20%
- 被平民弑王一次后，国王使用概率下降 40%

**典型台词触发**：
- 放弃大额金币时："太危险了……这轮让你。"
- 被迫出高阶棋子时："没办法了……只能拼一把。"

#### 激进型 AI

**核心策略**：追求最大金币收益，高风险高回报。

**行为特征**：
- 大额轮果断派出高阶棋子争夺
- 不害怕使用国王（即使对手可能有平民）
- 钻石使用偏早，倾向明牌升级（用声势碾压对手）
- 较少使用伪装（觉得没必要）
- 容易被"强装弱"陷阱反杀

**额外行为修正**：
- 大额轮时 F2（金币期望值）的权重临时上浮 40%
- F4（风险系数）的权重整体下降 50%
- 金币落后时，会进入"疯狂模式"——几乎所有因子权重让位于 F2
- 被反杀后不会变谨慎（这是它的弱点）

**典型台词触发**：
- 派出国王时："王驾亲临！"
- 被平民弑王时："不……不可能！"
- 大额轮获胜时："金币，都是我的。"

#### 欺诈型 AI

**核心策略**：通过信息不对称获利，擅长制造假象。

**行为特征**：
- 高频使用伪装（几乎每轮都伪装）
- 擅长"强装弱"——高阶棋子伪装低阶引诱对手出错
- 钻石偏好暗牌升级，制造信息差
- 会故意在小额轮打出某些棋子来塑造"人设"，然后在大额轮反转
- 读牌使用率高（获取更多对手信息来辅助欺诈）

**额外行为修正**：
- F7（伪装价值）在大额轮临时上浮 50%
- 暗牌升级的 `surpriseMultiplier` 额外 +0.3
- 会维护一个"对手对AI的认知模型"，故意做出反常行为打破对手预期
- 如果连续两轮出同类棋子，第三轮必定切换策略

**特殊机制——"设陷阱"连招**：

欺诈型 AI 会执行预设的多轮连招：

**连招 1："平民钓鱼"**
- 第 N 轮：出士兵，伪装成平民（塑造"我手里有平民"的假象）
- 第 N+1 轮（大额轮）：出真正的平民，暗升为二级平民，伪装成士兵
- 目标：引诱对手在大额轮出国王

**连招 2："威慑虚张"**
- 第 N 轮：明牌宣告使用钻石升级国王（但实际出的是普通国王，没升级）
- 第 N+1 轮：出真正的二级平民，伪装成普通平民
- 目标：利用上一轮的"钻石威慑"让对手以为 AI 钻石已用，放松警惕

**连招 3："连环平局"**
- 连续 2-3 轮主动制造平局
- 利用大额轮平局惩罚消耗对手高阶棋子
- 在对手高阶棋子耗尽后突然发力

#### 套路型 AI

**核心策略**：按预设的战术组合行动，复刻经典打法。

**行为特征**：
- 拥有 5-8 套预设战术组合（"剧本"）
- 每局开始时根据手牌和对手类型选择一套剧本
- 剧本执行过程中会根据局势微调，但大方向不变
- 如果剧本被打断（关键棋子被迫提前使用），切换到备用剧本

**预设剧本库**：

**剧本 A："钓鱼战术"**
1. 第 1-2 轮：出士兵，不伪装，制造保守假象
2. 第 3 轮（等待大额轮）：出平民暗升为二级平民，伪装成士兵
3. 目标：在大额轮完成平民弑王

**剧本 B："欺诈耗牌流"**
1. 大额轮：出高阶棋子伪装低阶，逼迫对手求平局
2. 利用平局惩罚消耗对手高阶棋子
3. 在对手高阶棋子耗尽后用剩余高阶棋子收割

**剧本 C："钻石骗招"**
1. 明牌宣告使用钻石，但实际派出普通弱棋
2. 对手被威慑，可能出保守棋子
3. 弱棋输给对手保守棋子，但保住了钻石和高阶棋子
4. 后续轮次用真正的高阶棋子 + 保留的钻石收割

**剧本 D："将军突击"**
1. 前期用士兵和骑士试探
2. 在大额轮果断出将军（不伪装）
3. 如果赢，后续用国王 + 钻石升级碾压
4. 如果平局（将军被封禁），切换到备用剧本 E

**剧本 E："残局收割"**（剧本 D 的备用）
1. 将军被封禁后，假装颓势
2. 引诱对手激进
3. 用平民弑王或暗升反击

### 4.3 人格对话系统

每种人格维护一个台词库，按事件类型分类，随机选取播放（避免重复）：

**事件类型**：
- `ROUND_START` — 回合开始
- `PLAY_PIECE` — 出牌
- `REVEAL_WIN` — 亮牌后 AI 赢
- `REVEAL_LOSE` — 亮牌后 AI 输
- `REVEAL_DRAW` — 亮牌后平局
- `DIAMOND_USE` — 使用钻石
- `DISGUISE_REVEALED` — 伪装被揭穿
- `KING_KILLED` — 国王被平民击杀（AI 方或被击杀方）
- `LARGE_ROUND` — 大额轮开始
- `GAME_WIN` — AI 赢得对局
- `GAME_LOSE` — AI 输掉对局
- `AMNESIA_TRIGGER` — 失忆 Debuff 触发

每个人格每个事件类型 3-5 句台词，随机选取，有冷却机制（同一句不连续出现）。

## 五、难度系统

### 5.1 三档难度的行为差异

难度不影响 AI 的"策略"，而是影响 AI 的"执行精度"。同一个欺诈型 AI，在入门难度下会因为噪声而偶尔犯错，在地狱难度下则几乎完美执行。

| 参数 | 入门 | 进阶 | 地狱 |
|------|------|------|------|
| 效用噪声范围 | ±0.30 | ±0.15 | ±0.05 |
| 伪装使用率修正 | ×0.3 | ×0.7 | ×1.0 |
| 钻石使用精度 | 随机时机 | 较优时机 | 最优时机 |
| 读牌使用精度 | 随机时机 | 较优时机 | 最优时机 |
| 对手模式学习深度 | 仅最近 2 轮 | 最近 4 轮 | 全部历史 |
| 连招执行概率 | 10% | 50% | 90% |
| 大额轮决策延迟 | 1-2秒 | 2-4秒 | 3-5秒 |

**噪声机制详解**：

噪声是在效用分数计算完成后、最终选择前加入的随机扰动：

```
finalScore(action) = U(action) + Random(-noiseRange, +noiseRange)
```

入门难度的 ±0.30 噪声意味着 AI 可能选到效用排名第 2-3 的行动（偶尔犯错），地狱难度的 ±0.05 意味着几乎总是选最优解。

**特殊情况**：即使在地狱难度下，AI 也不应该 100% 完美。保留微小的随机性，让 AI 偶尔可以被欺骗，否则玩家会感觉"不可能赢"。建议地狱难度的实际噪声保持在 ±0.03-0.05。

### 5.2 难度特有的行为限制

**入门难度**：
- AI 不使用暗牌升级（只用明牌升级或不用）
- AI 不使用"设陷阱"连招
- AI 在大额轮倾向保守（降低出高阶棋子的概率 30%）
- AI 伪装只使用简单的弱装强或强装弱，不做复杂伪装

**进阶难度**：
- AI 开始使用暗牌升级（概率 40%）
- AI 偶尔使用连招（概率 50%）
- AI 会根据金币差调整策略

**地狱难度**：
- AI 全面使用所有战术
- AI 会分析玩家的出牌模式并针对性反制
- AI 会执行多轮连招
- AI 会利用失忆 Debuff（在 Debuff 触发前刻意记住关键信息）

## 六、特殊决策模块

### 6.1 读牌决策模块

读牌是独立的二选一决策（使用/不使用），在进入棋子选择之前执行。

**决策流程**：

```
if (aiReadingsLeft > 0) {
    readValue = F9_read(uncertainty, roundGold, readingsLeft)
    if (readValue > READ_THRESHOLD[difficulty]) {
        useReading()
        // 获取信息：对手是否使用钻石？是否开启伪装？
        // 70% 真实，30% 误导
        updateBeliefs(readingResult)
    }
}
```

**读牌信息处理**：

读牌返回的信息不是确定性的。AI 需要维护一个"信念状态"（Belief State），对每个可能的对手行动维护概率估计。读牌信息会修正这些概率，但不会完全确定。

例如：读牌提示"对手使用了钻石"，AI 应该将对手使用钻石的概率从基础值（如 20%）提升到（如 65%），而非直接视为 100%。

**读牌阈值**：

| 难度 | READ_THRESHOLD |
|------|---------------|
| 入门 | 0.70（很少使用读牌） |
| 进阶 | 0.50（适度使用） |
| 地狱 | 0.35（积极使用） |

### 6.2 平民弑王决策模块

平民弑王是游戏中最关键的战术决策点。AI 需要专门评估"是否用平民去赌弑王"。

**触发条件**：AI 手中有平民 + 当前轮金币足够大（通常 ≥20）

**评估逻辑**：

```
kingKillValue = P(opponent_plays_king) × kingKillReward
              + (1 - P(opponent_plays_king)) × commonerLossCost
```

其中：
- `P(opponent_plays_king)` = AI 估算对手本轮出king的概率
- `kingKillReward` = 弑王成功的奖励（赢得大量金币 + 可能的绝杀翻盘）
- `commonerLossCost` = 平民白白损失的代价（失去一枚棋子 + 可能失去钻石升级）

**对手出king的概率估算关键因素**：
- 对手还有几枚棋子？如果只剩 1-2 枚且其中有国王，出king概率高
- 当前轮金币数？大额轮出king概率高
- 对手的历史行为？激进型人格出king概率高
- 对手是否还有钻石？如果有钻石，可能升级国王为二级国王

**二级平民 vs 二级国王的特殊评估**：

如果 AI 暗升平民为二级平民，且对手可能暗升国王为二级国王：
- 成功概率极高（二级平民是二级国王的唯一克星）
- 触发绝杀翻盘（直接清空对手所有金币，当场锁定胜利）
- 但需要同时满足：AI 有钻石 + 对手有国王 + 对手本轮出king + 对手升级了国王
- 联合概率很低，但收益极大

### 6.3 平局策略模块

AI 有时需要主动制造平局。

**主动求平局的触发条件**：
- 大额轮 + AI 手牌中没有把握赢的棋子
- AI 金币领先 + 想利用平局惩罚消耗对手高阶棋子（欺诈型/套路型专属）
- 小额轮 + 试探阶段，不想暴露棋子

**如何制造平局**：

AI 需要猜测对手可能出什么棋子，然后出同一阶级的棋子。

```
drawStrategy = {
    targetPiece: 猜测对手最可能出的棋子,
    aiPiece: 与 targetPiece 同阶级的 AI 棋子,
    disguise: 伪装成与 targetPiece 相同的棋子名（增强平局概率）
}
```

注意：平局不一定是"双方出完全相同的棋子"。规则中是"双方牌力相同"则平局。如果双方都伪装成同一个名称，但实际棋子不同，结算时按真实棋子比较——所以伪装只能骗对手改变出牌，不能直接制造平局。

修正：平局只有在双方真实棋子战力相同时才会发生。因此 AI 主动求平局需要精准预判对手出牌，然后出同阶棋子。

### 6.4 失忆 Debuff 应对模块

当失忆 Debuff 触发（全场剩余金币<40），双方各丢失 1 条历史出牌记录。

**AI 应对策略**：

1. **Debuff 触发前**：如果 AI 预判到即将触发失忆（根据剩余金币计算），地狱难度的 AI 会"提前记忆"——将关键对手行为模式编码到行为权重中，而不仅仅依赖历史记录
2. **Debuff 触发后**：AI 根据剩余的历史记录和已编码的行为权重继续决策，但信息完整度降低
3. **利用 Debuff**：欺诈型 AI 可能在 Debuff 触发前故意做出某种行为模式，期望 Debuff 恰好擦除对手对该模式的记忆，从而在 Debuff 后反转策略

## 七、对手行为模型（AI 如何"读懂"玩家）

### 7.1 行为追踪系统

AI 维护以下统计数据（每轮更新）：

```
OpponentModel {
    // 出牌倾向
    pieceUsageCount: { PieceType: int }       // 每种棋子使用次数
    avgPieceRankPlayed: float                  // 平均出牌战力
    largeRoundAvgRank: float                   // 大额轮平均出牌战力
    smallRoundAvgRank: float                   // 小额轮平均出牌战力

    // 伪装倾向
    totalDisguises: int                        // 总伪装次数
    disguisePatterns: { realType: { disguisedAs: int } }  // 伪装模式统计

    // 钻石倾向
    diamondUsedRound: int?                     // 在第几轮使用钻石（null=未使用）
    diamondUsageStyle: string                  // "open" / "secret" / "none"

    // 风险偏好
    riskSeekingScore: float                    // 0~1，越高越激进
    drawSeekingScore: float                    // 0~1，越高越倾向平局

    // 近期趋势（最近 3 轮）
    recentAggression: float                    // 近期激进度变化
}
```

### 7.2 出牌概率预测

基于上述模型，AI 为对手的每枚剩余棋子计算"本轮被出牌概率"：

```
P(opponent_plays_X) = baseWeight(X)
                     × goldModifier(X, currentRoundGold)
                     × personalityModifier(X, opponentRiskProfile)
                     × historyModifier(X, opponentModel)
                     / normalizationFactor
```

**各修正因子说明**：

- `baseWeight(X)`：基于棋子战力的基础权重（高阶棋子权重略高）
- `goldModifier`：大额轮提升高阶棋子权重，小额轮提升低阶棋子权重
- `personalityModifier`：如果对手表现出激进倾向，高阶棋子权重提升
- `historyModifier`：如果对手过去在大额轮频繁出某类棋子，该类权重提升

### 7.3 模型置信度

行为模型有一个"置信度"指标（0~1）：

- 前 2 轮数据不足，置信度低（<0.3），AI 主要依赖默认概率
- 3-5 轮后置信度中等（0.3-0.7），AI 开始利用行为模型
- 5 轮以上置信度高（>0.7），AI 高度依赖行为模型

失忆 Debuff 会将置信度强制降低 30-50%。

## 八、AI 对话系统

### 8.1 台词数据结构

```
DialogueLine {
    personality: AIPersonality
    eventType: DialogueEvent
    text: string
    condition: (GameState) => bool    // 可选的条件函数
    cooldown: int                     // 使用后冷却轮数
    priority: int                     // 优先级（高优先级覆盖低优先级）
}
```

### 8.2 台词选取逻辑

每轮检查是否有触发的事件，从符合条件的台词中随机选取一条（排除冷却中的台词）。同一事件多条台词时，优先选取优先级高的。

台词显示为对话气泡，出现在 AI 角色立绘旁边，持续 3-5 秒后淡出。

### 8.3 每个人格的台词风格

- **保守型**：语气温和、犹豫、自嘲，常用省略号和感叹号
- **激进型**：语气强硬、自信、傲慢，常用感叹号和反问句
- **欺诈型**：语气神秘、暗示、模棱两可，常用问号和省略号
- **套路型**：语气冷静、计算、预言式，常用句号（断言式）

## 九、性能与实现约束

### 9.1 性能要求

- AI 决策计算必须在 500ms 内完成（不阻塞主线程）
- 效用评估最多遍历 所有可用棋子 × 3种升级选项 × 5种伪装选项 = 最多约 75 个候选行动
- 对手概率估算使用查表法（预计算克制关系矩阵），避免每帧重复计算

### 9.2 可调试性

- 提供 AI 决策日志系统：每次决策记录所有候选行动的效用分数明细
- 日志格式应便于人类阅读（用于开发调试和直播解说）
- 提供"AI 思维可视化"选项（可在设置中开关）：在对局界面侧边栏实时显示 AI 正在考虑的选项和评分

### 9.3 可扩展性

- 权重配置应存储为 ScriptableObject 或 JSON 文件，方便调整平衡
- 新人格可以通过复制现有配置 + 调整权重来创建
- 新剧本可以通过配置化的方式添加（不需要改代码）
- 台词系统应支持本地化（文本走配置表）
